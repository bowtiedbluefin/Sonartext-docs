## Request Parameters

<ParamField body="input" type="string" required>
  The text to generate audio for. Maximum length is 4,096 characters.
  
  **Example**: `"Hello, this is a sample text for speech synthesis."`
</ParamField>

<ParamField body="model" type="string" default="tts-kokoro">
  TTS model to use for speech generation.
  
  **Available models**: `tts-kokoro` (currently the only supported model)
  **Example**: `"tts-kokoro"`
</ParamField>

<ParamField body="voice" type="string" default="af_sky">
  Voice to use for speech generation. Choose from 60+ available voices.
  
  **Categories**:
  - **af_** - Adult Female voices (American accent)
  - **am_** - Adult Male voices (American accent)  
  - **bf_** - British Female voices
  - **bm_** - British Male voices
  - **zf_** - Chinese Female voices
  - **zm_** - Chinese Male voices
  - **ff_** - French Female voices
  - **hf_** - Hindi Female voices
  - **hm_** - Hindi Male voices
  - **if_** - Italian Female voices
  - **im_** - Italian Male voices
  - **jf_** - Japanese Female voices
  - **jm_** - Japanese Male voices
  - **pf_** - Portuguese Female voices
  - **pm_** - Portuguese Male voices
  - **ef_** - Spanish Female voices
  - **em_** - Spanish Male voices
  
  **Popular options**: `af_sky`, `am_adam`, `bf_alice`, `bm_daniel`
</ParamField>

<ParamField body="response_format" type="string" default="mp3">
  Audio format for the output file.
  
  **Options**:
  - `mp3` - Most common, good compression
  - `wav` - Uncompressed, high quality
  - `flac` - Lossless compression
  - `aac` - Good compression, widely supported
  - `opus` - Modern codec, excellent for streaming
  - `pcm` - Raw audio data
</ParamField>

<ParamField body="speed" type="number" default="1.0">
  Speed of speech generation. Range: 0.25 to 4.0.
  
  **Examples**:
  - `0.5` - Half speed (slower)
  - `1.0` - Normal speed
  - `1.5` - 1.5x faster
  - `2.0` - Double speed
</ParamField>

<ParamField body="streaming" type="boolean" default="false">
  Whether to stream the audio response as it's generated.
  
  **Note**: Currently not implemented - responses are always buffered.
</ParamField>

## Example Usage

<CodeGroup>
```javascript JavaScript
async function generateSpeech(text, options = {}) {
  const response = await fetch('/v1/audio/speech', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${apiKey}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      input: text,
      model: options.model || 'tts-kokoro',
      voice: options.voice || 'af_sky',
      response_format: options.format || 'mp3',
      speed: options.speed || 1.0
    })
  });

  if (!response.ok) {
    throw new Error(`Speech generation failed: ${response.statusText}`);
  }

  // Return audio blob
  return response.blob();
}

// Generate speech and play it
const audioBlob = await generateSpeech(
  "Welcome to our text-to-speech API! This voice sounds natural and clear.",
  {
    voice: 'af_sky',
    speed: 1.2,
    format: 'mp3'
  }
);

// Create audio URL and play
const audioUrl = URL.createObjectURL(audioBlob);
const audio = new Audio(audioUrl);
audio.play();
```

```python Python
import requests
import io

def generate_speech(text, api_key, voice='af_sky', speed=1.0, format='mp3'):
    url = 'https://api.sonartext.com/v1/audio/speech'
    
    payload = {
        'input': text,
        'model': 'tts-kokoro',
        'voice': voice,
        'response_format': format,
        'speed': speed
    }
    
    headers = {
        'Authorization': f'Bearer {api_key}',
        'Content-Type': 'application/json'
    }
    
    response = requests.post(url, json=payload, headers=headers)
    
    if response.status_code == 200:
        return response.content  # Audio bytes
    else:
        raise Exception(f"Speech generation failed: {response.text}")

# Generate speech
text = "Hello! This is a demonstration of our text-to-speech capabilities."
audio_bytes = generate_speech(
    text=text,
    api_key='stx_live_your_api_key',
    voice='am_adam',
    speed=1.0,
    format='wav'
)

# Save to file
with open('generated_speech.wav', 'wb') as f:
    f.write(audio_bytes)

print(f"Generated {len(audio_bytes)} bytes of audio")
```

### Format Optimization

Choose the best format for your use case:

<AccordionGroup>
<Accordion title="üéµ MP3 - General Use">
  - **Best for**: Web applications, mobile apps
  - **Compression**: Good (smaller files)
  - **Quality**: High enough for most use cases
  - **Compatibility**: Excellent across all platforms
</Accordion>

<Accordion title="üéôÔ∏è WAV - High Quality">
  - **Best for**: Professional audio, editing
  - **Compression**: None (larger files)
  - **Quality**: Maximum fidelity
  - **Compatibility**: Universal support
</Accordion>

<Accordion title="üìª AAC - Mobile Optimized">
  - **Best for**: Mobile apps, streaming
  - **Compression**: Better than MP3
  - **Quality**: Excellent at lower bitrates
  - **Compatibility**: Great on mobile devices
</Accordion>

<Accordion title="üåê Opus - Web Streaming">
  - **Best for**: Real-time streaming, WebRTC
  - **Compression**: Excellent for low latency
  - **Quality**: Superior at low bitrates
  - **Compatibility**: Modern browsers
</Accordion>
</AccordionGroup>

## Billing & Usage

<Info>
Text-to-speech is billed per character processed. Pricing is based on your subscription plan.
</Info>

- **Character counting**: Includes all text characters, spaces, and punctuation
- **Minimum charge**: 1 character minimum per request
- **Rate limits**: Based on your plan (10-100 requests/minute)

## Best Practices

<AccordionGroup>
<Accordion title="üé≠ Voice Selection">
  - Test different voices to find the best match for your content
  - Consider your audience's language and accent preferences
  - Use appropriate gender voices for your use case
</Accordion>

<Accordion title="üìù Text Preparation">
  - Write text as you want it spoken (spell out numbers, abbreviations)
  - Use proper punctuation for natural pacing
  - Test with different sentence structures
</Accordion>

<Accordion title="‚ö° Performance">
  - Generate audio in advance when possible (not real-time)
  - Use appropriate audio formats for your platform
  - Implement proper error handling and retries
</Accordion>

<Accordion title="üîä Quality Control">
  - Test audio output with your target audience
  - Adjust speed based on content complexity
  - Consider the listening environment (headphones vs. speakers)
</Accordion>
</AccordionGroup>
